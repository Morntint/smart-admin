# AI 工具日期解析优化方案

## 问题背景

AI 模型没有实时时间感知能力，当用户说"查询今天的操作日志"时，模型可能会生成训练数据截止日期（如 2023 年），导致查询结果错误。

## 解决方案

### 1. 相对日期支持（核心优化）

在工具参数层面支持相对日期格式，让 AI 不需要知道具体日期，只需传递语义化的日期描述。

**支持的相对日期格式**：
- `today` - 今天
- `yesterday` - 昨天
- `this_week` - 本周（周一到周日）
- `this_month` - 本月（第一天到最后一天）
- `last_7_days` - 最近7天（含今天）
- `last_30_days` - 最近30天（含今天）
- `last_week` - 上周
- `last_month` - 上月

### 2. 智能日期解析器

文件：`app/admin/service/ai/DateParser.php`

```php
// 使用方式
$startDate = DateParser::parse($args['start_date'], 'start');
$endDate   = DateParser::parse($args['end_date'], 'end');
```

功能特性：
- ✅ 支持相对日期自动解析
- ✅ 自动校验年份合理性（过滤 2024 年以前的无效日期）
- ✅ 防止未来日期查询（自动修正为今天）
- ✅ 解析失败自动降级到今天
- ✅ 同时支持绝对日期 YYYY-MM-DD 格式

### 3. 系统提示词引导

在对话开始时自动给 AI 提示使用相对日期：

```
【重要】工具使用建议：
当用户查询"今天"、"本周"、"本月"、"最近7天"等时间相关内容时，请在调用工具时使用相对日期参数：
- "今天" / "今日" → start_date: "today"
- "昨天" → start_date: "yesterday"
...
这样可以避免因缺少实时时间感知而产生日期错误。
```

## 优势对比

### 旧方案（日期注入）
| 问题 | 说明 |
|------|------|
| 提示词膨胀 | 每次对话都要注入日期信息 |
| 模型理解偏差 | 模型不一定正确使用注入的日期 |
| 数据库查询 | 每次都要查提示词模板 |

### 新方案（相对日期）
| 优势 | 说明 |
|------|------|
| 语义化 | 模型更容易理解和正确使用 |
| 鲁棒性 | 即使模型用错，工具层也会自动纠正 |
| 性能好 | 无额外数据库查询 |
| 可扩展 | 方便添加更多相对日期格式 |

## 使用方法

1. **更新工具参数描述**：执行 `seed_query_log_tool.sql` 更新工具参数 schema
2. **测试对话**：发送"查询今天的操作日志"，观察 AI 调用工具时参数是否为 `start_date: "today"`
3. **验证结果**：检查返回数据是否为今天的日志

## 扩展建议

需要支持更多相对日期时，只需在 `DateParser::parse()` 中添加 case 即可：

```php
case 'last_quarter':
    // 实现上季度逻辑
    break;
```
