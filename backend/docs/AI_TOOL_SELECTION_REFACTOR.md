# AI 工具调用流程重构完成

## 概述

本次重构实现了**发送消息时选择工具**的功能，用户可以在每次发送消息时决定是否使用工具、使用哪些工具，而不是整个会话绑定固定的工具集合。

**设计原则：灵活、直观、用户可控**

---

## 核心设计

### 工具选择层级

| 层级 | 说明 | 实现方式 |
|------|------|----------|
| Agent 绑定 | Agent 管理员配置的可用工具 | 数据库关联关系 |
| 本次发送选择 | 用户发送消息时勾选的工具 | 接口参数 `tool_ids` |

### `tool_ids` 参数规则

| 参数值 | 说明 | 行为 |
|--------|------|------|
| **不传 / null** | 使用 Agent 绑定的所有工具 | AI 可以调用全部可用工具 |
| `[]` | 不使用任何工具 | 纯 AI 对话，无工具调用 |
| `[1, 2, 3]` | 使用指定 ID 的工具 | AI 只能调用用户勾选的工具 |

| GET | `/admin/ai/chat/agents/{id}/tools` | 获取 Agent 可用的工具列表 |

**请求示例**：

```json
{
  "conversation_id": 100,
  "content": "查询今天的操作日志",
  "tool_ids": [1, 2]    // 可选：不传=全部工具，[]=不用工具，[1,2]=用这两个
}
```

### 核心服务逻辑

**文件**：`app/admin/service/ai/AiConversationService.php`

方法 `getToolsToUse()` 实现筛选逻辑：

```php
private function getToolsToUse(array $agentTools, ?array $requestedToolIds): array
{
    // 1. 空数组 = 不使用任何工具
    if (is_array($requestedToolIds) && empty($requestedToolIds)) {
        return [];
    }

    // 2. 非空数组 = 只使用选中的工具
    if (is_array($requestedToolIds) && !empty($requestedToolIds)) {
        $selectedIdMap = array_flip($requestedToolIds);
        return array_filter($agentTools, function ($tool) use ($selectedIdMap) {
            return isset($selectedIdMap[$tool['id']]);
        });
    }

    // 3. null / 不传 = 使用全部
    return $agentTools;
}
```

## 前端变更

### UI 位置

工具选择器位于**输入框上方**，用户每次发送消息前都可以调整：

```
┌──────────────────────────────────────────┐
│ ☑ 启用工具    选择工具： [下拉多选 ▼]   │
├──────────────────────────────────────────┤
│ 输入消息，Enter 发送...                  │
└──────────────────────────────────────────┘
```

### 交互逻辑

1. **进入对话时**：加载该 Agent 的所有工具列表，默认全部勾选
2. **发送前**：用户可勾选/取消勾选工具，或直接关闭"启用工具"开关
3. **发送时**：根据当前勾选状态传递 `tool_ids` 参数
4. **灵活切换**：同一会话中，用户可以每次发送都切换工具配置

### 新增 API 方法

**文件**：`src/api/ai-manage.ts`

```typescript
// 获取 Agent 可用工具列表
fetchGetAgentTools(agentId: number)

// 流式发送（支持 tool_ids 参数）
fetchSendMessageStreamWithTools(data)
```

## 相对日期支持（解决 AI 日期错误问题）

### 问题背景

AI 模型训练数据有截止日期，不知道"今天"具体是哪一天，会错误地返回训练数据截止日期附近的数据。

### 解决方案

**文件**：`app/admin/service/ai/DateParser.php`

支持相对日期格式，AI 只需要传语义化的日期描述，服务端自动解析为真实日期：

| 相对日期 | 解析为 |
|----------|--------|
| `today` | 今天 |
| `yesterday` | 昨天 |
| `this_week` | 本周一 ~ 本周日 |
| `this_month` | 本月 1 号 ~ 本月最后一天 |
| `last_7_days` | 最近 7 天 |
| `last_30_days` | 最近 30 天 |
| `last_week` | 上一周 |
| `last_month` | 上一月 |

**同时做安全校验**：
- 自动纠正早于 2024 年的日期（AI 训练数据截止前）
- 自动纠正未来日期（超过今天的）

## 使用流程

```mermaid
flowchart TD
    A[用户进入对话] --> B[加载 Agent 绑定的工具列表]
    B --> C[默认勾选全部工具]
    C --> D[用户输入问题]
    D --> E{用户调整工具选择?}
    E -->|关闭工具开关| F[tool_ids = []，不使用工具]
    E -->|勾选部分工具| G[tool_ids = [选中的 ID]]
    E -->|保持默认全部| H[不传 tool_ids，使用全部]
    F --> I[发送消息]
    G --> I
    H --> I
    I --> J[AI 根据传入的工具决定是否调用]
```

## 优势对比

### 旧方案（会话绑定工具）

| 问题 | 说明 |
|------|------|
| ❌ 不灵活 | 整个会话只能用同一组工具 |
| ❌ 切换麻烦 | 想换工具需要开新会话 |
| ❌ 用户迷惑 | 用户不知道工具是"绑定"在会话上的 |

### 新方案（发送时选择工具）

| 优势 | 说明 |
|------|------|
| ✅ 极致灵活 | 同一会话内，每一条消息都可以用不同工具 |
| ✅ 直观易懂 | 用户明确知道这次发送会使用哪些工具 |
| ✅ 交互友好 | 工具选择就在输入框旁边，一眼可见 |
| ✅ 向后兼容 | 不传参数行为与之前完全一致 |
